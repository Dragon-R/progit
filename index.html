Comandos Importantes

Comando Comentario
$ git config --list Lista todas las configuraciones del usuario
$ git config --global core.editor code Configurar el editor de código predeterminado
$ git config --global user.name "John Doe" Configuración global Nombre de usuario Git
$ git config --global user.email johndoe@example.com Configuración global Email de usuario Git
$ git help comando o git comando --help Obtener ayuda relacionada a un comando
$ git init Inicializando un repositorio en un directorio existente
$ git add Especificar qué archivos quieres controlar
$ git commit -m "initial project version" Confirmar los cambios con un mensaje de referencia
$ git clone [url] Clonar un repositorio remoto
$ git clone [url] myname Clonar un repositorio remoto con nombre de carpeta diferente
$ git status Revisar el estado de los archivos del repositorio
$ git status -s Revisar el estado de los archivos del repositorio de forma abreviada
$ cat .gitignore Archivo Git ignore, para ingorar archivos no deseados en el proyecto final (.gitignore)
$ git diff Ver los cambios completos de cada archivo modificado
$ git diff --staged Ver lo que está preparado y será incluido en la próxima confirmación
$ git diff --cached Ver qué está preparado hasta ahora
$ git difftool Git Diff como Herramienta Externa, ver los cambios con programas
$ git commit Confirmar todos los cambios
$ git commit -v Incluirá en el editor el diff de los cambios completos que están confirmandos
$ git commit -m Escribir el mensaje de confirmación directamente en el comando commit
$ git commit -a -m "add message" Preparar automáticamente los archivos rastreados antes de confirmarlos
$ git rm filename Eliminar archivos de Git
$ git rm --cached filename Eliminar archivos de Git, conservando el archivo en el equipo
$ git rm -f filename Eliminar archivos de Git, eliminando el archivo en el equipo
$ git mv file_from file_to Renombrar un archivo desde git
$ git log Ver el Historial de Confirmaciones
$ git log -p Muestra las diferencias introducidas en cada confirmación
$ git log --graph Añade un pequeño gráfico ASCII mostrando tu historial de ramificaciones y uniones
$ git commit --amend Rehacer la última confirmación, modificar con los últimos cambios
$ git reset HEAD filename Sacar archivos del área de preparación
$ git checkout -- filename Descartar cambios hechos en un archivo
$ git remote Ver los remotos configurados
$ git remote -v Muestra las URLs que Git ha asociado al nombre y que serán usadas al leer y escribir en ese remoto
$ git remote add [nombre] [url] Añadir un remoto nuevo y asociarlo a un nombre para referenciar fácilmente
$ git fetch [nombre] Traer toda la información en un repositorio sin usar la URL
$ git pull Traer y combinar automáticamente la rama remota con la rama actual
$ git push [nombre-remoto] [nombre-rama] Enviar la rama master al servidor origin
$ git remote show [nombre-remoto] Ver más información acerca de un remoto en particular
$ git remote rename [nombre-actual] [nombre-nuevo] Cambiar el nombre de la referencia de un remoto
$ git remote rm Eliminar un remoto
$ git tag Listar las etiquetas disponibles
$ git tag -a [nombre] -m '[mensaje]' Crear una etiqueta anotada
$ git show [nombre-etiqueta] Muestra la información del etiquetador
$ git tag [nombre] Etiquetar un commit es mediante una etiqueta ligera
$ git tag -a [nombre] [checksum] Etiquetar commits mucho tiempo después, especificar el checksum (o parte de él)
git push origin [etiqueta] Transferir una etiqueta a los servidores remotos
$ git push origin --tags Transferir todas las etiquetas a los servidores remotos
$ git config --global alias.co checkout Esto significa que, en lugar de teclear git commit, solo se necesita teclear git ci.
